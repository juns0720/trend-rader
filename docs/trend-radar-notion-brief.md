# Trend Radar 기획 메모

## 한 줄 소개

SNS에서 막 퍼지기 시작한 트렌드를 발견하고, 크리에이터가 바로 콘텐츠로 만들 수 있게 타이밍·근거·촬영 각도까지 정리해주는 트렌드 큐레이션 서비스

## 주제

요즘 트렌드는 너무 빠르게 생기고 사라짐.
특히 릴스, 쇼츠, 틱톡처럼 짧은 영상 중심의 플랫폼에서는 “지금 뭘 찍어야 하는지”를 빠르게 판단하는 게 중요함.

Trend Radar는 단순히 인기 검색어를 보여주는 서비스가 아니라, SNS에서 재생산되고 있는 트렌드를 기준으로 지금 타도 되는 주제인지, 이미 늦은 주제인지, 어떤 각도로 콘텐츠화하면 좋은지까지 보여주는 서비스.

초기 목업은 음식, 디저트, 편의점, 카페, 팝업처럼 콘텐츠화가 쉬운 생활형 트렌드를 중심으로 구성.
이후에는 플레이스, 패션·뷰티, 라이프스타일, 밈·챌린지, 인물·콘텐츠 영역까지 확장 가능.

## 왜 해볼 만한 주제인지

- 트렌드는 많지만, 실제로 콘텐츠로 만들 수 있는 트렌드는 따로 있음
- 검색량만 높은 키워드는 이미 늦은 경우가 많고, SNS에서 막 재생산되기 시작한 신호가 더 중요함
- 소규모 크리에이터나 마케터는 매일 트렌드를 직접 찾고 검증하는 시간이 부담됨
- “이게 뜬다”보다 “왜 뜨고, 어떻게 찍으면 되는지”까지 주는 서비스가 더 실용적

## 타겟 사용자

### 1. 숏폼 크리에이터

- 매일 올릴 콘텐츠 소재가 필요한 사람
- 음식, 카페, 편의점, 챌린지, 장소 리뷰 콘텐츠를 만드는 사람
- 트렌드에 늦지 않게 올라타고 싶은 사람

### 2. 브랜드/마케팅 실무자

- SNS에서 어떤 소재가 반응을 얻는지 빠르게 보고 싶은 사람
- 신제품, 팝업, 캠페인 콘텐츠 아이디어를 찾는 사람
- 자사 브랜드와 연결할 수 있는 트렌드 신호를 모니터링하는 사람

### 3. 트렌드 탐색을 좋아하는 일반 사용자

- 지금 SNS에서 뭐가 뜨는지 보고 싶은 사람
- 유행하는 장소, 제품, 콘텐츠를 빠르게 소비하고 싶은 사람
- 트렌드에 대해 의견을 나누고 싶은 사람

## 핵심 기능

### 1. 오늘의 트렌드 카드

매일 아침 5~10개 정도의 트렌드를 카드 형태로 제공.

카드에 들어가는 정보:

- 트렌드 제목
- 카테고리
- 현재 타이밍
- 재생산량
- 24시간 증가량
- 간단한 설명
- 왜 뜨고 있는지
- 콘텐츠로 만들 수 있는 각도
- 첫 3초 훅 문장
- 플랫폼별 포맷 팁
- 주의사항
- 출처/근거 데이터

### 2. 트렌드 타이밍 표시

트렌드를 단순 인기순으로만 보여주지 않고, 지금 진입해도 괜찮은지 판단할 수 있게 표시.

- 선점 타이밍: 이제 막 퍼지는 중
- 서두르면 좋음: 한창 확산 중
- 이미 피크: 이미 많이 소비된 상태
- 지는 중: 회고형이나 반대 포지션이 아니면 비추천

이 부분이 서비스의 핵심 차별점.
사용자는 “인기 있다”가 아니라 “지금 내가 콘텐츠로 만들 가치가 있는지”를 판단할 수 있음.

### 3. 상세 트렌드 분석

카드를 누르면 더 깊은 분석 화면으로 이동.

상세 화면에서 보여줄 내용:

- 트렌드 단계
- SNS 재생산 콘텐츠 수
- Instagram/TikTok 재생산량
- 최근 24시간 증가량
- 네이버 검색 관심도 추이
- 전일 대비 변화
- 주요 검색층
- 관련 유튜브 영상
- 비슷한 검색어
- 같이 뜨는 해시태그

### 4. 검색과 북마크

- 키워드 검색
- 카테고리 필터
- 인기 검색어 바로 탐색
- 관심 있는 트렌드 북마크 저장

크리에이터가 나중에 찍을 소재를 따로 모아두는 용도.

### 5. 릴스/쇼츠형 트렌드 탐색

추가 기능으로, 트렌드 카드를 세로 숏폼 피드처럼 넘겨보는 탐색 방식.

기존 카드 리스트는 정보를 빠르게 비교하는 데 좋고, 숏폼 피드는 사용자가 더 가볍게 트렌드를 소비하는 데 좋음.

흐름:

1. 사용자가 세로 피드에 진입
2. 트렌드별 대표 이미지/영상/짧은 설명 표시
3. 위아래 스와이프로 다음 트렌드 탐색
4. 관심 있는 트렌드는 상세 분석으로 이동
5. 바로 북마크하거나 토론에 참여

이 기능은 일반 사용자에게도 접근성이 좋음.
기획상 “크리에이터 도구”에서 “트렌드 소비 플랫폼”으로 확장되는 지점.

### 6. 트렌드 토론 기능

숏폼 피드 중간중간 특정 트렌드에 대한 토론 콘텐츠를 삽입.

예시:

- 이 트렌드, 진짜 뜰까?
- 이 제품 가격 괜찮다 vs 비싸다
- 이 챌린지 재밌다 vs 억지다
- 지금 찍어도 된다 vs 이미 늦었다

사용자는 찬성/반대 중 하나를 선택하고, 짧은 의견을 남길 수 있음.

토론 기능에서 기대하는 효과:

- 단순 정보 소비에서 참여형 서비스로 확장
- 사용자 반응 자체가 트렌드 검증 데이터가 됨
- 어떤 트렌드가 호불호가 큰지 파악 가능
- 크리에이터는 댓글 반응을 보고 콘텐츠 각도를 더 잘 잡을 수 있음

## 사용자 플로우

### 일반 탐색 플로우

1. 앱 접속
2. 오늘의 트렌드 카드 확인
3. 카테고리별로 관심 트렌드 필터링
4. 카드에서 타이밍과 재생산량 확인
5. 상세 화면에서 근거 데이터와 콘텐츠 각도 확인
6. 찍을 만한 트렌드는 북마크

### 숏폼 탐색 플로우

1. 릴스/쇼츠형 피드 진입
2. 트렌드를 하나씩 넘겨보며 가볍게 탐색
3. 관심 있는 트렌드는 상세 분석 확인
4. 중간에 등장하는 토론 카드에서 찬반 선택
5. 다른 사람들의 의견 확인
6. 마음에 드는 트렌드는 저장

### 크리에이터 활용 플로우

1. 오늘 올라온 트렌드 확인
2. 선점 타이밍 또는 한창 퍼지는 중인 트렌드 중심으로 선택
3. 상세 화면에서 3초 훅, 촬영 각도, 플랫폼별 포맷 팁 확인
4. 주의사항과 출처 확인
5. 릴스/쇼츠/틱톡 콘텐츠 제작

## 관리자/운영 플로우

현재 목업에는 운영자용 콘솔도 포함.

운영자가 하는 일:

- 인플루언서 계정 확인
- 트렌드 후보 URL 등록
- LLM 자동 분석 결과 확인
- 키워드, 카테고리, 훅, 설명, 주의사항 수정
- 네이버 데이터랩, 네이버 검색, 유튜브 데이터로 자동 보강
- 검수 후 발행

관리자 화면 구성:

- 대시보드
- Quick Add 등록
- 검수 큐
- 인플루언서 보드
- API 사용량
- 주간 발행량
- 데이터 소스별 신호

이 구조를 보면 서비스는 완전 자동 추천보다는 “자동 수집 + LLM 보강 + 운영자 검수” 방식에 가까움.
초기 MVP에서 품질을 유지하기 좋은 방식.

## MVP 범위

초기 MVP는 모든 자동화를 완성하기보다, 서비스의 사용감과 정보 구조를 보여주는 데 초점.

MVP에 포함할 것:

- 오늘의 트렌드 카드
- 상세 분석 화면
- 카테고리 필터
- 검색
- 북마크
- 관리자 Quick Add
- 관리자 검수 큐
- 인플루언서 보드
- 더미 기반 트렌드 데이터

MVP 이후 추가할 것:

- 실제 API 연동
- 릴스/쇼츠형 탐색 피드
- 찬반 토론 카드
- 토론 댓글/의견 수집
- 사용자별 관심 카테고리 추천
- 트렌드 알림
- 크리에이터별 콘텐츠 제작 기록

## 서비스의 핵심 포인트

Trend Radar의 포인트는 “트렌드 데이터” 자체보다 “지금 이걸 어떻게 써먹을 수 있는지”에 있음.

비슷한 서비스가 인기 키워드나 검색량을 보여준다면, Trend Radar는 아래 질문에 답하는 쪽.

- 이 트렌드 지금 타도 되는지
- 왜 뜨고 있는지
- 누가 반응하고 있는지
- 어떤 플랫폼에서 잘 맞는지
- 어떤 식으로 찍으면 되는지
- 이미 늦었다면 어떤 반대 각도로 접근할 수 있는지
- 사람들은 이 트렌드에 찬성하는지, 반대하는지

## 앞으로의 방향

초기에는 크리에이터를 위한 트렌드 브리핑 도구로 시작.
이후 숏폼 탐색과 토론 기능이 붙으면, 사용자들이 직접 트렌드를 소비하고 판단하고 의견을 남기는 참여형 트렌드 플랫폼으로 확장 가능.

단순히 “요즘 뜨는 것 모음”이 아니라, 트렌드의 발생, 확산, 피크, 쇠퇴를 사용자와 함께 관찰하는 서비스 방향.

## 정리

Trend Radar는 빠르게 바뀌는 SNS 트렌드를 크리에이터와 마케터가 바로 활용할 수 있게 정리해주는 서비스.

현재 목업은 트렌드 카드, 상세 분석, 검색, 북마크, 관리자 콘솔까지 포함하고 있어서 서비스 구조를 보여주기 충분함.
추가로 릴스/쇼츠형 탐색 피드와 찬반 토론 기능을 붙이면, 정보형 도구에서 참여형 트렌드 플랫폼으로 확장할 수 있음.
